# PawPilot（原 Dog Office）项目体检报告

> 生成时间：本次体检 · 代码规模约 5100 行 · 构建/类型检查均通过

## 一句话定位

把你终端里一堆零散的 AI agent（Claude Code / Codex / 反重力 / Hermes…），
变成一间"看得见的办公室"：每个 agent 是一只狗坐在工位上，
你一眼就能看到谁在干活、谁在等你、谁报错了，并能在一个 PM 的指挥下
让它们像一个团队那样分工协作完成一个完整项目。

---

## 一、现在能做什么（已落地的能力）

### 1. 单 agent 可视化监控（最成熟，可日常用）
- 任意终端 agent 用 `dog run` 包一层，自动出现在办公室成为一只狗
- 实时显示：当前在干什么、进度条、运行时长、6 种状态
  （工作中 / 思考中 / 摸鱼 / 等你回应 / 报错 / 失联）
- 工位卡片是 Marvis 风格的 3D 等距视角（浅色、弥散阴影、椅背挡身只露狗头）
- 点工位看详情：实时日志、token/耗时统计、归属关系
- 每个工位一个"打开终端"按钮：一键把对应终端窗口拉到最前
  （Terminal/iTerm 用窗口标题 + tty 双重定位，最准）
- 详情面板可直接打字发到对应终端（剪贴板+粘贴+回车，发完焦点自动回来）
- 桌面通知：完成/报错/等回应会弹，点通知跳回并选中那只狗（去重，不轰炸）
- 完成的狗会自己走出办公室归档；档案柜可翻历史

### 2. 多 Agent 团队协作（Phase 1 后端已通，前端 UI 待补）
- **9 种角色**，每个都有精心写的 System Prompt（resources/roles/）：
  PM、前端、后端、全栈、质检、测试、UI 设计、文档、DevOps
- **PM 指挥模式**：PM 拆任务 → 写接口契约 → 等你审批 → 自动招人派活
- **团队通信命令**（dog CLI 已实现并冒烟通过）：
  - `dog team` 看在岗成员
  - `dog send <名> "<话>"` 点对点喊话队友
  - `dog submit "<说明>"` 交付，进入"待验收"挂起（不下班，不烧配额）
  - `dog escalate "<病历>"` 质检红线上报 PM
  - `dog hire <角色> --task "..."` PM 招人（校验只有 PM 能招）
- **归属关系自动建立**：PM 派的狗自动认 PM 当爹（环境变量继承，PM 零负担）
- **例外管理**：质检默认沉默，只在"同一问题第 3 次 / 累计 3 轮"红线时惊动 PM
- **待验收生命周期**：交付≠下班，质检通过或你手动放人才走；被打回原地返工
- **模型不写死**：每个角色默认工具/模型只是默认值，招聘时可改，跟你 Claude Code 配置走
- **多项目并行**：数据层已支持项目维度，每个项目独立团队

### 3. 工程化
- macOS .dmg 打包就绪（`npm run dist`），有专属狗头图标
- 单实例锁（重复打开自动聚焦已有窗口，不再端口冲突崩溃）
- 老数据库自动迁移、SQLite 持久化
- 应用重启时把"上次还在跑"的狗标记为"中断"，可生成接手上下文恢复（handoff）

## 二、能解决哪些任务

| 场景 | 怎么帮你 |
|---|---|
| 同时开 5-10 个终端跑 agent，看不过来 | 一屏看全部状态，谁卡了一眼看到 |
| 不知道某个 agent 跑到哪了 | 工位实时显示当前动作 + 进度 |
| agent 在等你批准但你切走了 | "等你回应"状态 + 桌面通知叫你 |
| 满屏终端找不到对应窗口 | 工位点一下直接跳过去 |
| 想做个完整项目（不只单点任务） | PM 拆活、派给前后端/质检分工协作 |
| 项目跑一半关了电脑 | 重启后标记中断，可生成 handoff 续上 |

## 四、最大的缺口 & 下一步方向

### 🥇 第一优先：前端 UI 还停在"2 角色"时代（Phase 2）
**后端已经全面支持 9 角色 + 项目 + 待验收 + 中断恢复，但界面没跟上：**
- 招聘弹窗还只有"主管/员工"两个选项，点不出 9 个角色
- 没有项目标签栏（多项目并行但界面切不了项目）
- 没有"新建项目选文件夹"的入口
- "待验收 / 中断"状态的狗没有专属视觉姿态
- 没有验收面板（PM 终验报告 + 你点"通过/打回"的按钮）
- 没有 handoff 恢复的入口按钮

> 这是把"能力"变成"你点得到"的关键一步，工作量中等，但偏视觉，
> 建议交互式做（你历来要逐版审视觉）。

### 🥈 第二优先：真实跑一遍完整项目，验证协作闭环
目前团队协作只做过命令级冒烟测试，没用真实 Claude Code/Codex
跑过一个端到端项目。建议拿个小项目（如登录页）实测，
看 PM 拆活、契约先行、质检打回、待验收这套流程在真实模型上跑不跑得顺，
prompt 哪里要调。

### 🥉 第三优先：体验打磨
- 每个工具（Codex/反重力）的模型/Key 配置（之前讨论过，暂缓）
- 扫描已在裸跑的 agent 收编进办公室
- 团队解散动画、返工轮次角标等氛围细节
- 配额用量估算提示（避免并行太多打爆订阅窗口）

## 六、Phase 2 前端落地（本轮新增，已实测）

把后端早已支持、但界面点不到的能力全部接上了：

- **招聘弹窗重做为 9 角色九宫格**：PM / 前端 / 后端 / 全栈 / 质检 / 测试 /

- **新建项目流程**：＋新项目 → 选文件夹 → 自动以文件夹名建项目并切过去；
  该项目招的 PM/成员终端自动 cd 到项目目录、归属该项目
- **统计栏加"待验收"**，PM 独占"主管室"行、员工在"员工区"
- **品牌更名 PawPilot**（标题）
- **接手上下文/复制日志** 按钮加了"✓ 已复制"反馈
- 顺手修了一个 UX 隐患：空办公室原本整块是点击区，易误触弹出招聘——已改为只有明确按钮可点
