## 0. 一句话

PawPilot（原名 Dog Office）：把终端里的多个 AI agent（Claude Code/Codex 等）变成一间"看得见的办公室"，每个 agent 是一只狗坐在工位上；并支持一个 PM 带队、9 角色分工完成完整项目（Vibe Coding 协作）。

## 1. 项目位置 & 启动

- 路径：`/Users/trystan/Desktop/文件管理项目/Vibe Coding/dog-office`
- 启动：`cd` 进去 → `npm run dev`（Electron + Vite）
- 健康检查：`curl -s --noproxy '*' http://127.0.0.1:17890/health`
- 打包：`npm run dist`（macOS arm64 dmg）
- 技术栈：Electron + React + TypeScript + Zustand + Tailwind + better-sqlite3 + Express/WS；CLI 是 Python 脚本 `sdk/dog`

## 2. 关键环境事实（务必记住，踩过坑）

- **app 数据目录**：`~/Library/Application Support/pawpilot/dog-office.db`（注意：文件夹叫 pawpilot 因为 package.json name 改成了 pawpilot；db 文件名仍是 dog-office.db）。旧数据在 `~/Library/Application Support/dog-office/`，已被改名孤立（数据没丢，想找回就复制旧 db 覆盖）。
- **桌面 TCC 限制**：项目在 ~/Desktop 下，macOS 隐私保护使终端里的 python 读不了桌面下的脚本。已解决：dog 脚本启动时自动安装到 `~/.dot-office/bin/dog`（非保护目录），从那跑。角色提示词落地在 `~/.dog-office/prompts/`，启动脚本在 `~/.dog-office/launch/`。
- **单实例锁**：app 同时只能跑一个；重复启动会聚焦已有实例。改主进程代码后要彻底 `pkill -9 -f electron-vite` + `pkill -9 -f "dog-office/node_modules/electron"` + 杀 17890 端口，再重启，否则旧实例还在跑你测的是旧代码。
- **辅助功能权限没授权**：当前"发消息到终端"靠 osascript 模拟键盘，需要 macOS 辅助功能权限，用户没授 → 发消息失败。下一步要改掉这个机制（见第 5 节第 1 条）。
- **Claude Code 登录**：用户终端的 claude 已登录订阅（Sonnet 4.6 · Claude Pro）。之前没登录导致 agent 起来不回话，已解决。注意桌面 app 内置 claude(2.1.181) 与 npm 全局 claude(2.1.186) 共用 `~/.claude/.credentials.json`，token 轮换可能互相挤掉线 → 401，重新 `/login` 解决。
- **🔴 代理透传（国内必踩，团队跑不通的真主因）**：用户在国内，直连 Anthropic 会被 **403 Request not allowed**（按 IP 地区封锁）。用户靠本地 Clash(`127.0.0.1:7890`) 手动 `export ...proxy...` 才能用。但 dog 在**全新 Terminal 窗口**起，不继承这个手动代理 → 狗直连 → 403 → 一直“不回话”。**已修**：`terminal.ts` 的 `resolveProxyExports()` 把代理写进每只狗的启动脚本；来源 ① `~/.dog-office/proxy.env`（已建，存着用户的 7890 代理）② PawPilot 进程环境。改代理只改这个文件即可，新狗自动生效。症状辨认：401=登录问题，403=代理/地区问题。
- 验证 Electron 应用：用 `npm run dev` 后台跑 + `screencapture -x -o /tmp/x.png` 截图 + Read 看图；不要用 preview_start（那是浏览器 preview，不适用 Electron）。
- sdk/dog 兼容老 python：文件顶部有 `from __future__ import annotations`（系统 python 3.9 不支持 `str | None`）。

## 3. 架构分层

- `src/main/db.ts` — SQLite：agents 表（含 role/status/projectId/parentId/terminalTty/cwd/command 等）+ projects 表 + logs 表；markStaleAgentsInterrupted、buildHandoff
- `src/main/server.ts` — Express+WS 端口 17890：/api/events（started/log/status/submitted/escalation/completed…）、/api/team、/api/send、/api/hire；心跳检测；killAgent
- `src/main/terminal.ts` — 招聘起终端：拼角色提示词、写启动 sh 脚本、osascript 拉起 Terminal/iTerm/Warp；sendToTerminal/focusTerminal（osascript 键盘，待改）；resolveDogBin（装 dog 到 home）
- `src/main/roles.ts` — 9 角色注册表（id/name/emoji/duty/defaultTool/defaultModel）
- `src/main/index.ts` — 主进程入口：窗口、通知、单实例锁、IPC handlers（hire/send/focus/list-roles/项目管理/get-handoff…）
- `sdk/dog`（Python）— 包装器：dog run（PTY 包子进程、心跳、解析输出成"干净活动流"）；子命令 dog team/send/submit/escalate/hire；环境变量继承建立归属
- `resources/roles/*.md` — 9 份角色 System Prompt + _common.md（共 ~2.2 万字），启动时拼接 _common + 角色文件注入
- 渲染端 `src/renderer/src/`：App、HeaderBar（项目标签栏）、OfficeFloor、Workstation（3D 工位 SVG）、DetailPanel（详情+活动流+对话框）、HireModal（9 角色）、store/agentStore
